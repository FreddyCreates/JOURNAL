🔧 Verification Modes

🔍 Verify

Full verification pipeline — extract claims, map evidence, assess posture, generate sealed packets.

thesis verify .

🗺️ Evidence Map

Map all implementations. Detect stubs, dead code, and function completeness across the repo.

thesis evidence-map ./src

📋 Claims

Extract and classify every research claim. Proven, implemented, hypothetical, aspirational, overstated.

thesis claims ./docs

🔗 Lineage

Trace code provenance. Map dependency chains, creation order, and evolution over time.

thesis lineage .

📦 Packet Build

Generate integrity-sealed verification packets in JSON or markdown. Publication-ready bundles.

thesis packet build .

📰 Preprint Split

Split monolithic verification into preprint-sized series for staged publication.

thesis preprint split .

🔐 Manifest

Generate SHA-256 hash manifests for all artifacts. Deterministic, tamper-evident integrity.

thesis manifest .

🛡️ Seal

Create cryptographic verification seals with chain lineage. Links to previous attestations.

thesis seal .

⚖️ Design Principles

🚫 Zero Hallucination

Only reports what is evidenced in code, tests, and outputs. Never invents claims.

🔄 Deterministic

Same input → same verification output, always. Reproducible by design.

🔐 Hash-Sealed

Every packet cryptographically attested with SHA-256 manifests and chain seals.

🧩 Mode-Based

Each capability is an independent, composable mode. Use one or all.

🏗️ CI-Native

Runs in any pipeline without human intervention. GitHub Actions ready.

💻 Local-First

No cloud dependency for core verification. Terminal-native, Julia-powered.
